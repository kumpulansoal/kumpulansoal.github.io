17. Buat program interface, yg di dalamnya ada abstract class dan abstrac method, silakan anda buat programnya secara bebas.


Jawaban:

Class abstrak adalah class yang masih dalam bentuk abstrak. Karena bentuknya masih abstrak, dia tidak bisa dibuat langsung menjadi objek.

Sebuah class agar dapat disebut class abstrak setidaknya memiliki satu atau lebih method abstrak.

Method abstrak adalah method yang tidak memiliki implementasi atau tidak ada bentuk konkritnya.

Bingung?

Gini aja deh:

Method abstrak itu adalah method yang tidak memiliki isi.

Cuma ada nama saja..

// ini abstrak method

void sayHello();

// ini bukan abstrak method karena

// punya implementasi di body method

void greeting(){

System.out.println("Hello Java");

}

..dan class yang menggunakan method ini, secara otomatis akan menjadi class abstrak.

Mengapa Harus Pakai Class Abstrak?

Kita tahu, class abstrak memang belum bisa digunakan secara langsung.

Karena itu, agar class abstrak dapat digunakan, maka ia harus dibaut bentuk konkritnya.

Cara membuat class abstrak menjadi konkrit adalah dengan membuat implementasi dari method-method yang masih abstrak.

Ini bisa kita lakukan dengan pewarisan (inheritance).

Class abstrak biasanya digunakan sebagai class induk dari class-class yang lain. Class anak akan membuat versi konkrit dari class abstrak.

Lalu pertanyaanya:

Mengapa sih class harus dibuat menjadi abstrak?

Kan bisa saja kita pakai class yang biasa.

Iya, bisa-bisa saja.

Tapi pada suatu kondisi tertentu, class induk tidak ingin kita buat sebagai objek.. karena kode methodnya belum jelas mau diimplementasikan seperti apa.

Maka class ini sebaiknya dijadikan abstrak.

Biar makin paham, coba perhatikan contoh berikut:

class Database {

String getTableName(){

return null;

}

}

class MySQLDatabase extends Database {

@override

String getTableName(){

return this.sql("SELECT table_name FROM database.tabel")

}

}

Kedua class tersebut meripakan class biasa. Sekarang mari kita coba buat objek dengan class Database.

23. Apabila kita ingin membuat kelas baru java dengan nama Mahasiswa.java dimana merupakan kelas turunan dari kelas java Manusia.java, maka harus ditulis keyword… A. Public class Mahasiswa implement Manusia B. public class Mahasiswa implements Manusia C. public class Mahasiswa extend Manusia D. public class Mahasiswa extends Manusia


Jawaban:

D. public class Mahasiswa extends Manusia

Penjelasan:

Kata kunci extends digunakan untuk mendeklarasikan kelas turunan (subclass) dari kelas atasnya (superclass). Dengan kata kunci extends, kelas turunan akan mewarisi data field dan metoda yang dimiliki oleh kelas di atasnya.

Sebenarnya tidak semua data field dan metoda diwariskan kepada kelas turunnannya. Data field dan metoda yang dideklarasikan dengan modifier private, tidak diwariskan kepada kelas turunannya. Berikut ini adalah sintaksis untuk mendeklarasikan kelas turunan (subclass) dari superclass:

public class nama-subclass extends nama-superclass {

  // data dan metoda

}

31. pemrograman berorientasi objek Java soal: dapatkah kita membuat class di dalam sebuah class? jelaskan cara membuatnya jika hal tersebut dapat dilakukan​


Jawaban:

Ya, dalam pemrograman berorientasi objek Java, kita dapat membuat class di dalam class lainnya. Konsep ini dikenal sebagai "nested class" atau "inner class". Inner class adalah class yang dideklarasikan di dalam class lainnya.

Untuk membuat inner class di Java, kita perlu mengikuti sintaksis berikut:

```java

class OuterClass {

// Anggap ini adalah class utama

// Deklarasikan inner class di dalam class utama

class InnerClass {

// Isi dari inner class

}

}

```

Perhatikan bahwa deklarasi inner class berada di dalam kurung kurawal class utama. Inner class dapat mengakses semua anggota (atribut dan metode) dari class utama, termasuk anggota yang bersifat pribadi (private).

Ketika ingin menggunakan inner class, kita perlu membuat instance dari inner class dengan menggunakan instance dari class utama. Contoh penggunaan inner class seperti di bawah ini:

```java

class OuterClass {

private int outerData;

class InnerClass {

private int innerData;

public void display() {

System.out.println("Outer data: " + outerData);

System.out.println("Inner data: " + innerData);

}

}

}

public class Main {

public static void main(String[] args) {

OuterClass outerObj = new OuterClass();

OuterClass.InnerClass innerObj = outerObj.new InnerClass();

innerObj.display();

}

}

```

Dalam contoh di atas, kita membuat instance `outerObj` dari class `OuterClass`, kemudian menggunakan instance tersebut untuk membuat instance `innerObj` dari inner class `InnerClass`. Setelah itu, kita dapat memanggil metode `display()` dari inner class untuk mengakses data dari class utama dan inner class.

Ini adalah cara dasar untuk membuat dan menggunakan inner class di Java. Inner class sering digunakan untuk mengorganisir kode yang saling terkait secara logis dan membatasi akses ke class tertentu.

35. Tuliskan aturan penting dalam penulisan class pada Java​


Adapun aturan penting dalam penulisan class pada Java, antara lain:

1. Nama class dimulai dengan huruf kapital dan ditulis dalam gaya "UpperCamelCase".

Contoh: MyClass, StudentData, OrderManagement

2. Class harus memiliki method utama (main method) dengan tipe data "public static void main(String[] args)".

Metode ini berguna untuk menjalankan program Java.

6. Method harus ditulis dalam gaya "lowerCamelCase" dan harus diawali dengan kata kerja (verb). Contoh: getData(), calculateAverage()

8. Hindari menggunakan huruf besar untuk nama variabel dan method. Hal ini menyulitkan membaca kode pada saat pengembangan program.

9. Gunakan indentasi dengan benar pada setiap level blok program untuk memudahkan pembacaan kode.

10. Selalu periksa sintaks pada saat membuat class. Kompilator Java hanya mengenali kode yang ditulis dengan benar.
