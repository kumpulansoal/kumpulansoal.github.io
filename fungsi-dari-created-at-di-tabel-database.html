13. Membuat database apotek yang terdiri dari tabel obat, tabel pasien dan tabel transaksi kemudian buatlah struktur field Dan berikan primery key gambarlah


Pembahasan:

 

Database apotek yang terdiri dari tabel obat, tabel pasien dan tabel transaksi dengan struktur field dan primary key adalah:

 

1.    Tabel obat

 

Nama field: id_obat

Jenis data: integer / number

Keterangan field: primary key

 

Nama field: nama_obat

Jenis data: text / varchar

 

Nama field: pembuat_obat

Jenis data: text / varchar

 

Nama field: stok_obat

Jenis data: integer / number

 

Nama field: tanggal_kadaluwarsa

Jenis data: date

 

2.    Tabel pasien

 

Nama field: id_pasien

Jenis data: integer / number

Keterangan field: primary key

 

Nama field: nama_ pasien

Jenis data: text / varchar

 

Nama field: tanggal_lahir_pasien

Jenis data: date

 

3.    Tabel transaksi

 

Nama field: id_transaksi

Jenis data: integer / number

Keterangan field: primary key

 

Nama field: id_ pasien

Jenis data: integer / number

Keterangan field: foreign key, dengan mengambil data dari tabel pasien

 

Nama field: id_obat

Jenis data: integer / number

Keterangan field: foreign key, dengan mengambil data dari tabel pasien

 

Nama field: jumlah_transaksi

Jenis data: integer / number

 

Dalah database ini, tiga tabel ini saling berhubungan (lihat diagram di bawah).  Setiap transaksi pembelia obat, dicatat dalam table transaksi.

 

Nama obat dan pembeli (nama pasien) tidak dicatat, karena ada kemungkinan ada obat atau pasien dengan nama sama. Sehingga yang dicatat ada id_pasien dan id_obat, yang berupa foreign key.

 

Selain id_transaksi sebagai primary key ini, serta id_pasien dan id_obat, juga dalam tabel ini dicatat jumlah_transaksi untuk jumlah obat yang dibeli.

Mata Pelajaran: Teknologi Informasi

Materi: Database

Kata Kunci:  Perancangan Database, Primary Key, Foreign Key

16. cara menghapus database dan tabel (pake MYSQL)


Menghapus Tabel
DROP TABLE [IF EXIST]  table_name [, table_name, ...]
keterangan : parameter [IF_EXIST] berupa opsional yang berguna menghilangkan pesan error jika tabel tidak ada.
cara pakai : mysql> Drop Table [IF EXIST]  tabel_mahasiswa;

Menghapus Database
DROP DATABASE [IF_EXIST] Database_name;
Keterangan : parameter [IF_EXIST] berupa opsional yang berguna menghilangkan pesan error jika database tidak ada.
cara pakai : mysql> DROP DATABASE [IF_EXIST] DB_Penjualan;

31. Cara menggunakan fungsi statistik database


Praktikum_6 (Fungsi STATISTIK DATABASE © Ari Wibowo 200921FUNGSI STATISTIK DATABASE Apa Fungsi Statistik Database?Adalah suatu fungsi yang antara lain digunakan untuk menghitung, menjumlah atau mencari nilai tertentu pada suatu database dengan menggunakan syarat-syarat (criteria) tertentu. Fungsi ini berbeda dengan fungsi statistik yang telah dibahas sebelumnya. Perbedaanya adalah jika pada fungsi statistik biasa tidak menggunakan syarat atau kriteria tertentu. Rumus Statistik Database1.Fungsi DSUM Rumus: =DSUM(database,field,criteria) 2.Fungsi DCOUNT Rumus: = DCOUNT(database,field,criteria) 3.Fungsi DMAX Rumus: = DMAX(database,field,criteria) 4.Fungsi DMIN Rumus: = DMIN(database,field,criteria) 5.Fungsi DAVERAGE Rumus: = DAVERAGE(database,field,criteria) 6.Fungsi SUMIF Rumus: = SUMIF (range,criteria, [sum_range]) 7.Fungsi COUNTIF Rumus: = COUNTIF(range,criteria) Keterangan: database : tabel databasenya (termasuk judul), field : nomor kolom yang dikenai fungsi, dan kriteria : tabel syarat Keterangan: range : jangkauan criteria : syarat [sum_range] : jangkauan yang dijumlahkan
di gunakan untuk berhitung atau lain sebagainya seperti layaknya calculator ataupun alvalink
