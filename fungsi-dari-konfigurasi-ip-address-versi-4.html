7. 4. Jelaskan langkah-langkah konfigurasi mapping dari nama domain www.buku.net ke IP Address 192.168.11.1!


Langkah-langkah konfigurasi mapping untuk mengarahkan sebuah domain ke alamat IP tertentu bisa kita pisahan menjadi 2 tahapan yang merupakan bagian integral dari sistem yanng kita sebut dengan DNS (Domain Name System). Adapun langkah tersebut adalah terdiri dari menentukan name server lalu membuat DNS Record.

Penjelasan:

Menentukan Name Server

Name Server adalah sebuah server yang berisi database (basis data) yang menyimpan informasi tentang "kemana sebuah domain harus diarahkan" yang dalam hal ini dari nama domain ke alamat IP.

Setiap registrar (penyedia alamat domain) pada umumnya sudah menyertakan name server sendiri. Karena dalam pertanyaan ini kamu tidak menyertakan name server maka kita asumsikan saja bahwa domain tersebut sudah memiliki name server (sudah terkonfigurasi)

Menambah A Record

Anggap saja domain yang kamu miliki sudah tersimpan dallam sebuh name server. Nah, langkah selanjutnya adalah menambahkan instruksi untuk mengarahkan domain/subdomain ke alamat IP tertentu dengan cara membuat DNS Record.

DNS Record ini sendiri banyak macamnya, ada A record, CAA record, CNAMES, catatan MX record, dan TXT dimana setiap record tadi memiliki fungsi masing-masing.

Adapun yang akan kita gunakan kali ini adalah A record yang pada dasarnya memiliki fungsi sebagai catatan yang menyimpan domain/subdomain ke alamat IP tertentu.

Pada umumnya A record ini terdiri dari 2 buah catatan yaitu host dan Adress Berikut langkahnya:

Host akan kita isi dengan awalan dari nama domain. Dalam kasus ini bisa kamu ini dengan www

Adress adalah alamat IP dari server yang digunakan. Dalam kasus ini bisa kamu isi dengan 192.168.11.1

15. berapa jumlah oktal biner dalam satu rangkaian ip address versi 4 ​


Jawaban:

Sebuah alamat IP versi 4 terdiri dari empat blok angka desimal yang mewakili 32 bit. Setiap blok angka tersebut dapat diwakili dalam bentuk oktal (basis 8) atau biner (basis 2). Jadi, dalam satu rangkaian alamat IP versi 4, terdapat empat blok oktal dan empat blok biner, masing-masing mewakili 32 bit total.

21. berapa jumlah oktal biner dalam satu rangkaian ip address versi 4​


Jawaban:

Dalam satu rangkaian alamat IP versi 4 (IPv4), terdapat empat bagian oktal yang disebut oktet. Setiap oktet direpresentasikan dalam bentuk bilangan biner dengan panjang 8 bit. Jadi, jumlah total bit dalam alamat IP versi 4 adalah 4 oktet * 8 bit/oktet = 32 bit.

Sebagai contoh, alamat IP versi 4 "192.168.0.1" dapat diuraikan menjadi biner sebagai berikut:

- 192: 11000000

- 168: 10101000

- 0: 00000000

- 1: 00000001

Total bitnya adalah 32 (8 bit + 8 bit + 8 bit + 8 bit).

34. Ip address versi 4 (ipv4) merupakan kumpulan bilangan biner yang terdiri dari…


Jawaban & Penjelasan:

Alamat IP terdiri dari 32 bit.

Alamat IP terdiri dari 32 bit, sering ditampilkan sebagai 4 oktet angka dari 0-255 yang direpresentasikan dalam bentuk desimal, bukan dalam bentuk biner. Misalnya, alamat IP: 168.212.226.204 dalam bentuk biner adalah 10101000.11010100.11100010.11001100.

Internet Protocol versi 4 (IPv4) mendefinisikan alamat IP sebagai angka 32-bit. Namun, karena pertumbuhan Internet dan menipisnya alamat IPv4 yang tersedia, versi baru IP (IPv6), menggunakan 128 bit untuk alamat IP, dikembangkan pada tahun 1995, dan distandarisasi pada bulan Desember 1998.

Ukuran awalan, dalam bit, ditulis setelah miring. Ini disebut "notasi garis miring". Ada total 32 bit dalam ruang alamat IPv4. Misalnya, jika sebuah jaringan memiliki alamat "192.0.2.0/24", angka "24" mengacu pada berapa banyak bit yang terkandung dalam jaringan. Ini berarti alamat host tunggal. Itu disebut notasi CIDR. 192.168.0.1 mewakili IP dan /32 mewakili jumlah bit dalam mask.

39. cara konfigurasi ip address di debian 9


Untuk melihat Interface yang tersedia pada Komputer (Server Debian), gunakan perintah :

ip a

￼

Terlihat bahaw terdapat 2 Interface, yaitu enp0s3 dan enp0s8. Setiap komputer penamaanya tidak sama, jadi pastikan dahulu nama Interface nya sebelum melangkah ke tahap berikutnya. Untuk melakukan pengaturan IP Address, edit File /etc/network/interfaces dengan perintah berikut :

nano /etc/network/interfaces

lalu sesuaikan dengan kondisi jaringan￼

Simpan dan Keluar

Lalu untuk konfigurasi DNS Server, edit File /etc/resolv.conf  dengan perintah berikut :

nano /etc/resolv.conf

masukan DNS Servernya￼

Simpan dan Keluar

untuk mejalankan konfigurasi IP Address diatas, maka gunakan perintah :

systemctl restart networking

Cek apakah sudah terkoneksi dengan melakukan Pinging ke alamat IP atau alamat Web tertentu. Jika Reply maka konfigurasi IP Address telah berhasil.￼
