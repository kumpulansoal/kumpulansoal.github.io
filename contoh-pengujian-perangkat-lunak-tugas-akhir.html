17. sebutkan jenis jenis lisensi perangkat lunak! jelaskan satu persatu! [Soal Ujian Akhir Semester UAS 2015]


Lisensi perangkat lunak terdiri dari :

Lisensi Commercial.Lisensi TrialLisensi Non-CommercialLisensi SharewareLisensi FreewareLisensi Royalty-Free Binaries

Pembahasan

1. Lisensi Komersial

Adalah homogen lisensi yang biasa ditemui dalam aplikasi misalnya Microsoft menggunakan OS Windows & Officenya, Lotus, Oracle & sebagainya. Software yang diciptakan menggunakan lisensi ini yang memang dibentuk buat kepentingan komersial sebagai akibatnya user yang ingin menggunakannya wajib membeli atau menerima biar  penggunaan menurut pemegang hak cipta (Copyright).

2. Lisensi Trial Sofware

Adalah jenis lisensi yang biasa ditemui dalam aplikasi buat keperluan demo menurut sebuah aplikasi sebelum diluncurkan ke rakyat atau umumnya telah diluncurkan namun mempunyai batas masa aktif. Lisensi ini mengijinkan pengguna buat menggunakan, menyalin atau menggandakan aplikasi tadi secara bebas. Tetapi lantaran bersifat demo, tak jarang aplikasi menggunakan lisensi ini tidak mempunyai fungsi & fasilitas selengkap versi komersialnya. Biasanya dibatasi sang masa aktif eksklusif.

3. Lisensi Non Komersial Use

Biasanya diperuntukan buat kalangan pendidikan atau yayasan eksklusif pada bidang sosial. Sifatnya yang non komersial, umumnya perdeo namun menggunakan batasan penggunaan eksklusif.

4. Lisensi Shareware

Mengijinkan pemakainya buat menggunakan, menyalin atau menggandakan tanpa wajib meminta ijin pemegang hak cipta (Copyright). Berbeda menggunakan trial Software, lisensi ini  dibatasi batas saat masa aktif & mempunyai fitur yang lengkap. Lisensi jenis ini umumnya ditemui dalam aplikasi perusahaan kecil.

5. Lisensi Freeware

Biasanya ditemui dalam aplikasi yang bersifat mendukung, menaruh fasilitas tambahan, misalnya plug in yang melekat dalam aplikasi induk misalnya Eye Candy yang melekat dalam aplikasi Proprietary Adobe Photoshop.

6. Lisensi Royalty-Free Binaries

Serupa menggunakan lisensi freeware, hanya saja produk yang ditawarkan merupakan library yang berfungsi melengkapi aplikasi yang telah terdapat & bukan adalah suatu aplikasi yang berdiri sendiri.

7. Lisensi Open Source

Membebaskan usernya buat menjalankan, menggandakan, menyebarluaskan, mempelajari, mengubah, & menaikkan kinerja aplikasi. Berbagai jenis lisensi open source berkembang sinkron menggunakan kebutuhan, contohnya GNU/GPL, The FreeBSD, The MPL. Jenis-jenis aplikasi yang menggunakan lisensi ini contohnya Linux, Sendmail, apache, & FreeBSD.

18. Berikanlah contoh dari hubungan dan interaksi antara perangkat lunak dengan perangkat lunak​


Jawaban:

Penjelasan:

**Contoh hubungan dan interaksi antara perangkat lunak dengan perangkat lunak:**

* **Komunikasi:** Perangkat lunak dapat saling berkomunikasi satu sama lain untuk berbagi data atau informasi. Contoh:

   * Web browser berkomunikasi dengan server web untuk menampilkan halaman web.

   * Aplikasi email berkomunikasi dengan server email untuk mengirim dan menerima email.

   * Aplikasi media streaming berkomunikasi dengan server media untuk memutar video atau musik.

* **Kolaborasi:** Perangkat lunak dapat bekerja sama untuk menyelesaikan tugas yang kompleks. Contoh:

   * Sistem operasi menggunakan driver perangkat keras untuk mengontrol perangkat keras komputer.

   * Aplikasi produktivitas menggunakan komponen perangkat lunak lainnya untuk menampilkan grafik, merender teks, atau memproses data.

   * Aplikasi permainan menggunakan mesin permainan untuk menghasilkan grafis dan animasi.

* **Interkoneksi:** Perangkat lunak dapat terhubung satu sama lain untuk membentuk sistem yang lebih besar. Contoh:

   * Sistem operasi Windows terdiri dari banyak komponen perangkat lunak yang saling berhubungan.

   * Sistem manajemen database terdiri dari server database, klien database, dan aplikasi database.

   * Sistem e-commerce terdiri dari server web, aplikasi web, dan aplikasi seluler.

Berikut adalah beberapa contoh spesifik dari hubungan dan interaksi antara perangkat lunak dengan perangkat lunak:

* **Perangkat lunak aplikasi berkomunikasi dengan sistem operasi:** Perangkat lunak aplikasi menggunakan sistem operasi untuk mengakses sumber daya komputer, seperti memori, CPU, dan perangkat keras. Misalnya, aplikasi pengolah kata menggunakan sistem operasi untuk menampilkan teks di layar dan mencetak dokumen.

* **Perangkat lunak aplikasi berkomunikasi dengan perangkat lunak aplikasi lain:** Perangkat lunak aplikasi dapat berkomunikasi satu sama lain untuk berbagi data atau informasi. Misalnya, aplikasi email dapat berkomunikasi dengan aplikasi kalender untuk menambahkan acara ke kalender.

* **Perangkat lunak aplikasi berkomunikasi dengan perangkat keras:** Perangkat lunak aplikasi dapat berkomunikasi dengan perangkat keras untuk mengontrol perangkat tersebut. Misalnya, aplikasi permainan dapat berkomunikasi dengan kartu grafis untuk menampilkan grafis.

Hubungan dan interaksi antara perangkat lunak dengan perangkat lunak menjadi semakin penting seiring dengan berkembangnya teknologi. Perangkat lunak yang saling berhubungan dan dapat bekerja sama dapat memberikan pengalaman pengguna yang lebih baik dan efisiensi yang lebih tinggi.

=> Perangkat lunak

-Perangkat lunak adalah koleksi dari berbagai informasi dari intruksi dan data untuk memberitau komputer bagaimana cara bekerjanya.

-Fungsi dari perangkat lunak inilah adalah untuk memproses data yang sudah di-input oleh perangkat keras untuk memberikan hasil yang diminta. Bagian yang berperan di sini adalah CPU yang merupakan otak dari komputer.

-Contoh dari perangkat lunak adalah perangkat lunak aplikasi, perangkat lunak pemrograman, perangkat lunak utility, dll.

=>Pengertian Perangkat Lunak

Perangkat lunak juga disebut sebagai sistem komputer yang tidak memiliki wujud fisik yang diinstal dalam sebuah komputer atau laptop agar bisa dioperasikan, contohnya adalah Microsoft word, Microsoft excel, Microsoft power point, dan masih banyak lagi.

39. terangkan pengertian pengujian perangkat lunak??​


Jawaban:

Pengujian perangkat lunak (software testing) adalah bagian integral dari sebuah pembangunan perangkat lunak (software development). Sering kali, pembangunan sebuah perangkat lunak disempitkan dengan pengkodean (coding) atau pemograman dan pengujian perangkat lunak adalah salah satu proses vital dalam pembangunan perangkat lunak yang sering diabaikan. Padahal proses ini sangat penting, terutama untuk pembangunan perangkat lunak yang akan diproduksi secara masal (mass production), yang menuntut adanya jaminan mutu dan bebas kesalahan (bug-free) sebelum masuk pada tahap produksi. Meskipun begitu, untuk pembangunan perangkat lunak berdiri sendiri (standalone) atau dibangun untuk keperluan tertentu (in-house development), proses pengujian akan dapat menurunkan secara signifikan biaya pembangunan (development cost) dan biaya perawatan (maintenance cost).
