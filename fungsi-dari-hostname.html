1.Protokol adalah sistem peraturan yang memungkinkan terjadinya hubungan, komunikasi, dan perpindahan data antara dua komputer atau lebih. ... Sederhananya, protokol adalah media yang digunakan untuk menghubungkan pengirim dan penerima.

contoh fungsi protokol

Addressing adalah Header IP paket mengandung alamat yang memberikan identifikasi ke komputer pengirim dan penerima. Router menggunakan informasi ini untuk menuntun setiap paket melewati network komunikasi dan menghubungkan antara komputer pengirim dan penerima.

Timeouts adalah Setiap IP paket mengandung self-destructive counter yang membatasi umur dari paket. Jika paket sudah kadaluarsa, paket dihancurkan sehingga jaringan internet tidak mengalami overloaded dengan paket yang rusak.

2. menurut saya bisa Ya dan bisa TDK karena WAN dan Man adalah memiliki pencakupan area yg lebih luas sehingga jika wan dan man menggunakan topologi yg di pake kan kemungkinan untuk bus dan ring mungkin akan berkerja tapi untuk start menurut saya TDK akan bisa karena start Jika jarak antar komputer terlalu jauh ke komputer lain maka harus dibantu dengan repeater.

3.IP Address (Internet Protocol Address) adalah nomor biner atau identitas numerik yang yang dipakai disetiap komputer agar komputer tersebut dapat saling berhubungan. ... IPv4 atau IP versi 4 adalah IP yang memiliki panjang angka 32 bit sedangkan IPv6 atau IP versi 6 adalah IP yang memiliki panjang angka 128 bit

4.Alamat IP atau network address adalah alamat yang berbentuk angka yang mengidentifikasi setiap mesin di Internet. Oleh karena itu, setiap mesin memiliki alamat IP yang unik. Sebagai contoh, alamat IP website perusahaan abc bisa 192.228.180.200.

Sebuah nama domain adalah bentuk tekstual alamat yang lebih mudah bagi manusia untuk mengingat daripada rangkaian angka yang membuat sebuah alamat IP. Sebagai contoh, nama domain dari perusahaan abc yang sama bisa.

35. Aplikasi server yang bertanggung jawab dalam proses penerjemahan hostname menjadi IP address adalah..??


Aplikasi server yang bertanggung jawab dalam proses penerjemahan hostname menjadi ip address adalah Domain Name System. DNS sendiri merupakan sistem yang mirip dengan buku telepon dalam kehiduopan sehari - hari. Dengan DNS, penerjemahan host name menjadi ip adress dilakukan dengan pemanggilan dalam direktori yang ada pada database.

Pembahasan

Cara kerja Domain Name System terdiri dari beberapa tahap diantaranya :

Resolving nameserverRoot nameserverTLD nameserverAuthoritative nameserverRecords

Tahap resolving nameserver merupakan tahap dimana terjadi pemanggilan query oleh client sehinggqa server harus dapat menyediakan informasi yang dibutuhkan. Proses ini dapat melibatkan beberapa server lebih sebagai bantuan.

Tahap rooting nameserver ini menerima permintaan dari nameserver resolver lalu meneruskannya kepada TLD untuk mencari informasi secara lebih spesifik

TLD nameserver merupakan tahap dimana para ekstensi domain umum seperti .com disimpan dan diatur. TLD akan menerima permintaan dari root nameserver untuk selanjutnya diarahkan pada authoritative nameserver.

Authoritative nameserver merupakan tahap dimana permintaan infromasi akan diberikan seputar nama domain yang ada dan menyediakan akses pada ip address yang disiman dalam records
